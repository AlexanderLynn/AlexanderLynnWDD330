Errors, Exceptions, Warnings

Errors are normally caused by one or more of 3 things: System error, the actual system that the program
is interacting with failed, a Programmer error, meanign faulty logic or syntax was created and tried to be
executed, or User error, meaning data was entered incorrectly aor unhandlable by the executing program. It is
a best practice to limit what data can be input into our systems or causing the entry to conform as it is hard to
control external systems.

An exception can also be thrown when executing code. These are caused by a variable or function that cannot
be returned. Thankfully with most exceptions in Javascript, stack tracing is available. This is the way Javascript
can trace back where an exception or error may have originated. A list of the entire execution or is dislayed
along with the line and sometimes column of the issue.

Warning displayed in Javascript do not stop the execution of the code. While this could be good for allowing
the remaining code to continue to function, this could be problematic was it is not always very evident when these
warnings occur and the terminal is not open.

Debugging and Console Tools

Due to the constant flux of api support between Javascript, CSS, and HTML, Javascript natively support a feature
called Feature Detection. This is used to check to see if the current version of Javascript being ran by the browser
supports the api trying to be called. Implementing this can help reduce errors of the entire Javascript ceasing to
run. The program simply skips the method or contained code if the api isn't present.

All modern browsers contain at least a basic set of debugging software. This allows us to set breakpoints within
the codes execution. One handy way to do this is with the alert() method. When called, it behaves similarly
to the console.log() method in that is pushes a string or variables value to the console, but it also pauses
the execution of the code and allows for debugging. While inside of this console the command console.trace() can
also be used to provide an interactive stack track, as earlier mentioned. THis can aid in the debugging process or more
complex solutions.

An Error is a default object type in Javacript. It can be passed in a custom response or console dialog
With using a custom error object, we are able to combine it with the throw statement to aid in our debugging.
When the throw statement is executed it stops all execution of Javascript code and displays the error using the object
previously created. The program then is terminated. Also helpful when debugging and working with throw statements are
the keywords try, catch,, and finally. These statements can be used to test code and try running more
code instead of simply ending the execution.

Testing

In all programming lanagues, not just Javascript, it is important to test code. It is even possible to create methods
for running simulations of what inputs would be expected. Using this kind of development is know as TDD or Test Driven
Development. Within the TDD methods there are 5 steps:

  1. Write tests (that initially fail)
  2. Write code to pass the tests
  3. Refactor the code
  4. Test refactored code
  5. Write more tests for new features
