Javascript Objects

Javascript is an object oriented langauge. Programming with objects is also known as OOP or
object oriented programming.
Objects are defined to represent entities. These entities can not only have name:value pair properties
assigned to them, but also methods. These methods are functions callable wihin the object. This is most
often seen in the format of objectName.method();

Inside of these methods, there is a special keyword that can be used to refer to the current object
in run-time. Often the this takes the place of the object which is calling the method (The
exception is with arrow functions which simply refer back to the outer context or container that
called the arrow function). Using the this keyword can be very powerful for increasing re-
usability of your code.

To help clean up code and ensure that variable names do not overlap and possibly overrite eachother,
we can defines namespaces with an object literal pattern. This means creating a const objects and placing
functions and variables specific to that program within the object. The functions or variables would then
need to be preceeded by the object literals name to be invoked. One popular way is to create JSON files.

JSON or Javascript Object Notation is used as a lightweight way to store data. In fact, Javascript even
has methods built-in to handle the reading and writing of these objects. The parse() method will
read a string and create a Javascript Object. The stringify() method does the opposite. It turns a
Javascript object into a string. Note; stringify() ignores methods within the object.

Javascript also features the usage of constructors for objects. These are useful for creating an object
with predetermined methods and variables, some even populated by default until overridden. For example,
if you have a list of cars, you could create a constructor for the object type of car and during the
creation or afterwards, populate the attributres of say, color, horsepower and weight. These could be used
with a method int hat object type to determine the max speed of the vehicle. (Red is +2 max speed for instance)

The Document Object Model

The Document Object Model or DOM is a heirarchical way of showing the relationship between elements in
an HTML document. This page is almost fully mutable during runtime and post load of the page via scripting.
The most common ways of sleecting these document objects, which correspond to the HTML tags is by looking up
their id, class name, tag name, or through a query selector, the later using CSS notation for DOM selection.

Query selectors are very powerful in the way that you can refine more indepthly your DOM selection as opposed
to a simple tag or id. For example, the statement const hero2 = document.querySelector("ul#roster:nth-child(2)");
would select the 2nd child of the unordered list who's class is "roster"

Many powerful methods are built into navigating the DOM such as .parent, .firstChild, or .nextSibling

Methods for interacting and changing element attributes and properties as well as adding and removing
elements are also baked into Javascript. This including re-ordering of elements and updating CSS properties
in code during run-time as well.

Javascript Events

Some of the most basic and powerful events within Javascript are event listeners. Event listeners can be
added onto HTML elements for things such as clicks, load, mouseover, and many more. When the event listener
is triggered, the callback function is called. Not only mouse events but keyboard and touch events can be
registered. This gives us a way to even further get code out of or HTML documents and into Javascript files.
In the same way that attributes can be added or removed in the DOM using Javascript, so can event listeners.
Event listeners can even be used to stop or prevent default behavior.

One of the most important concepts to understand when using event listeners is understanding how the event
is propagated. There are two ways they do so: bubbling and capturing.
The default behavior is bubbling. This means that when an event is registered, the same event will be
propagated to all parent elements. For example, if you click on an <li> element, the ul, body,
and html elements will all also register the event. The propagation can be stopped by including the.stopPropagation();
method within the event being handled.
